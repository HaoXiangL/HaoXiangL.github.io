# Mutation

在Redux的思想定义中，唯一可以改变state的只能通过dispatch提交一个action。

在Vuex中，Mutation对应action的功能，唯一可以改变state的方法是 提交 Mutation。Vuex 中的 mutation 非常类似于事件：每个 mutation 都有一个字符串的 事件类型 (type) 和 一个 回调函数 (handler)。

TIP

这个回调函数就是我们实际进行状态更改的地方，并且它会接受 state 作为第一个参数

export default new Vuex.Store({
state: {
  count: 1
},
mutations: {
  // add 就是它的事件类型(type)
  add(state){
    //变更state
    state.count ++
  }
})

注意，你无法像之前一样直接调用this.$store.mutations.add去调用这个回调函数，Mutation里的回调函数类似于 声明一个事件，你需要用特定的方法去触发它，这个方法就是store.commit()，对应redux里的dispatch。它接收的参数就是你在Mutation里 声明的 事件类型 (type)

store.commit('add')

//组件中使用
this.$store.commit(type: String)

# 提交载荷（Payload）

你可以向 store.commit 传入额外的参数，即 mutation 的 载荷（payload）：

  mutations: {
  add(state,payload){
    state.count += payload
  }
}

注意，当你选择payload为一个普通类型时，需要考虑类型之间的转换。上面的payload参数如果不传参，会导致计算出错(结果为NaN)

但是官方推荐payload为对象为大多数情况，这样可以包含多个字段并且记录的 mutation 会更易读：

mutations: {
  add (state, payload) {
    state.count += payload.amount
  }

# 对象风格的提交方式

提交 mutation 的另一种方式是直接使用包含 type 属性的对象：

store.commit({
  type: 'add',
  amount: 10
})

当使用对象风格的提交方式，整个对象都作为载荷传给 mutation 函数，因此 handler 保持不变：

  add (state, payload) {
    state.count += payload.amount
  }

# 注意事项

不难发现，Vuex中的状态是响应式的，维持响应式应遵循Vue里的规则。

  • 如果要建立响应式的State，需要提前在Store中定义(类似在Data中定义响应式数据)
  • 如果需要在老对象中添加新属性，应该：
    • 使用 Vue.set(obj, 'newProp', 123), 或者
    • 利用对象展开运算符,如下：
    state.obj = { ...state.obj, newProp: 123 }
    1

# Mutation 必须是同步函数

mutation是记录前后状态快照的一个依据，如果使用异步回调函数，在回调还没发生前，mutation已经触发，这样就会导致前后状态快照的紊乱，无法确定是否已经更变state
