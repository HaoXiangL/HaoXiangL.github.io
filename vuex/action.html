# Action

action是用作提交异步改变store的一个方法，定义上与mutation相同。

不同的是:

  • action提交的是一个mutation
  • action可以进行异步操作
export default new Vuex.Store({
state: {
  count: 1,
},
mutations: {
  add(state,payload){
    state.count += payload
  }
},
actions: {
  actionAdd(context){
    context.commit('add')
  }
}
})

Action接受一个类似store的对象(context)，因此你可以像store.commit('mutationsKey')一样，使用context.commit()方法去提交一个mutation，但它并不是store本身。

建议在编写action时使用参数解构的方式去引入，这样可以方便操作多次commit

actions: {
    actionAdd({context}){
      commit('add')
    }
  }

# 如何触发Action

官网将调用action的方法定义为 分发，使用store.dispatch触发。

由于mutation限制必须使用同步执行方式，Action为了补充这种不足，我们可以在action中不受限制的使用 异步操作。

actions: {
actionAdd ({ commit }) {
  setTimeout(() => {
    commit('add')
  }, 1000)
}
}

# 在组件中触发Action

跟mutation相似，在组件中也是通过this.$store.dispatch('xxx')或使用 辅助函数 mapActions去触发。

import { mapActions } from 'vuex'

export default {
  // ...
  methods: {
    ...mapActions([
      'actionAdd', // 将 `this.actionAdd()` 映射为 `this.$store.dispatch('actionAdd')`

      // `mapActions` 也支持载荷：
      'incrementBy' // 将 `this.incrementBy(amount)` 映射为 `this.$store.dispatch('incrementBy', amount)`
    ]),
    ...mapActions({
      add: 'actionAdd' // 将 `this.add()` 映射为 `this.$store.dispatch('increment')`
    })
  }
}

# 组合 Action

Action是一个异步的操作，dispatch可以处理action的回调函数返回的promise对象，而且它(store.dispatch)仍然会返回一个promise对象。

这个官方的解释有点绕，我的理解是：你可以在action里定义promise对象给dispatch，dispatch可以处理，然后我也是返回promise对象给你。

actions: {
  actionA ({ commit }) {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        commit('someMutation')
        resolve()
      }, 1000)
    })
  }
}

actionA返回一个promise对象，dispatch在接受这个action时，把它自身返回值也变成promise对象，因此你可以使用promise中的then和catch去处理异步操作。

也可以在action中返回另一个action

actions: {
// ...
  actionB ({ dispatch, commit }) {
    return dispatch('actionA').then(() => {
      commit('xxx')
    })
  }
}

// 假设 getData() 和 getOtherData() 返回的是 Promise

  async actionA ({ commit }) {
    commit('gotData', await getData())
  },
  async actionB ({ dispatch, commit }) {
    await dispatch('actionA') // 等待 actionA 完成
    commit('gotOtherData', await getOtherData())
  }
}
