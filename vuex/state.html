# State

Vuex 使用单一状态树 这是基础的概念,只有一个store以保证状态管理工具的唯一性。

# 在 Vue 组件中获得 Vuex 状态

由于 Vuex 的状态存储是响应式的，从 store 实例中读取状态最简单的方法就是在 计算属性中返回某个状态：

store/index.js

import Vue from 'vue'
import Vuex from 'Vuex'

Vue.use(Vuex)

export default new Vuex.Store({
state: {
  count: 0
},
mutations: {
  add(state){
    state.count ++
  }
}
})

不要忘记在 main.js中注入store选项。从根组件添加store可以把store实例注入到所有子组件。 （需调用 Vue.use(Vuex)）

src/main.js

import Vue from 'vue'
import App from './App.vue'
import store from './store'

Vue.config.productionTip = false

new Vue({
  store,
  render: h => h(App)
}).$mount('#app')

App.vue

   <h1>{{ count }}</h1>

<script>
export default {
  name: 'app',
  computed: {
    count() {
      return this.$store.state.count

  },

# mapState 辅助函数

如果只在computed属性中定义方法去获取Vuex的state，未免会太过麻烦和冗余了，mapState就是从这个痛点出发去优化获取Vuex状态的方式。

<script>
import { mapState } from 'vuex'

export default {
computed: mapState({
  //写法1
  count: state => state.count,

  //写法2
  count: 'count',

  // 写法3
  count (state) {
    return state.count
  }
})
}
</script>

mapStae提供了3种不同的写法，方便大家去使用。写法1采用对象方法模式，配合箭头函数。写法2采用键值对模式，传入的字符串为Store中定义的State，等同于方法1的箭头函数。方法三不可以使用箭头函数，会导致this丢失。

当所在Vue组件的computed名称与store中所定义的state名称相同时，我们可以传入一个字符串数组

computed: mapState([
  'count'
])

# 对象扩展运算符

mapState函数返回一个对象，如果需要合并到computed中，扩展运算符...能够很方便的做到。

computed: {
  //其他的computed方法
  localComputed(){

  },
  ...mapState([
    'count',
  ])
},
